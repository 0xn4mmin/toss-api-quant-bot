# 투자 지식 대장 (Investment Knowledge Base)

이 봇에 "심어진" 투자 지식의 전체 목록이다. 각 항목은 **근거(문헌·논리) → 코드
위치 → 검증 방식**으로 추적된다. 여기 없는 지식은 봇에 없는 것이고, 새 지식은
이 문서와 STRAT 스펙 개정 + 사전등록 + 게이트를 거쳐서만 들어온다.

> 원칙: **넓게 아는 것보다 정확하게 아는 것.** 항목 수를 늘리는 것 자체는
> 목표가 아니다 — 각 항목이 다중검정을 통제한 채 검증 가능해야 한다.

## 1. 수익의 원천 (알파 가설)

| 지식 | 근거 | 코드 | 상태 |
|---|---|---|---|
| 횡단면 모멘텀 (12−1, 스킵 포함) | Jegadeesh-Titman(1993) 이래 자산군·시대 횡단 재현 — 가장 견고한 이상현상 | `slots/trend_score.py` | 게이트 판정 대기 |
| 절대 모멘텀 필터 (mom>0만) | Antonacci(2014) — 하락장 노출 차단 | `trend_score.apply_abs_filter` | 그리드 탐색 대상(on/off) |
| 자산군 듀얼 모멘텀 + 현금 도피 | Antonacci — 상대×절대, 미달 시 현금이 MDD 방어의 핵심 | `slots/dual_momentum.py` (v1.4) | draft — 별도 id·별도 사전등록 |
| KR 외국인·기관 수급 강도×지속성 | 정보 우위·자금 지속성 가설 — **학술 근거 얇음**, tossctl 고유 데이터 엣지 | `slots/kr_flows.py` | paper 격리 (표본 3년 전 합격 유보, OF-01) |

## 4. 검증 방법론 (이 봇의 진짜 자산)

| 지식 | 근거 | 코드 |
|---|---|---|
| 사전등록 — 탐색 공간을 먼저 봉인 | 다중검정 통제 (Harvey et al. — 발견된 팩터 대부분은 데이터 마이닝) | `prereg.py` + append-only registry |
| OOS는 전략 id당 1회 | "여러 번 본 OOS는 OOS가 아니다" | `judge.evaluate_oos` |

## 6. 기대치의 눈금

이 계열 전략의 정직한 성공 기준: **세후 CAGR 한 자릿수 후반~10%대 초반,
MDD ≤ 15%**. 백테스트가 연 수십%를 보이면 축하가 아니라 과적합·편향을
의심하라 — 이 봇의 게이트는 그런 결과를 탈락시키기 위해 존재한다.
